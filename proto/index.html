Proto

AppTools on Github.

This is the first section of a prototyping tool that allows a person to leverage their knowledge of html and css, without needing extensive knowledge of javascript (or any at all) to create a mostly working prototype of a website or app. The goal is not to allow the complete creation of a working application (although it certainly could be done) but to provide a basis for production that can more easily be transitioned into actual development than most current offerings.

The problem with most current prototyping implementations is that they often take a good amount of work that must all be completely redone once actual work is put forward. With HamProto a working viable prototype can be provided to a development team and a lot of css and html will have already been created, greatly reducing the transition period between prototype and bootstrapping.

Jumps

Proto Jumps

Proto uses sections to define different app "pages". Each section of a Proto app has its own unique id, and so linking to any of them is as easy as making a .proto-jump link to it. These can be any type of element, as long as it has a .proto-jump class hook, and an href attribute that links to a #id.

Both of the buttons below work using the .proto-jump hook, even though one is an anchor and one is a button. Using your own styles will help you design things however you want them to look, but anything can have a .proto-jump added to it, even this phrase (click me).

Dynamic content can be added to and updated on pages using the "pageshow" event attached to the document element.

Lists

Lists

Lists are a common element in webpages and apps alike. A simple implementation allows maximum flexibility for the user to design their own lists, while giving a basic infrastructure and ideas for creation.

With expandable list items you can create dynamic lists, and using the .proto-toggle-next and .proto-toggle-parentnext hooks you can open and close items from a couple of common locations.

  • List Item

    Creating expandable list items is pretty straight forward, using the proto-toggle-next hook. This element begins open, using the .proto-nextopen hook, but can still be toggled. Whereas the next list item's togglable element begins closed.

Accordions

A common type of list is an accordion. This type of list will only allow one element to be open at a time. Use the .proto-list-accordion hook around your accordion items, and then implement the .proto-toggle-nextaccordion or .proto-toggle-parentnextaccordion hooks to show or hide a .proto-togglable element.

A Header for all occasions

The headers provided with Proto are quite versatile although they are mostly aesthetic. The real value added comes in their combination with a body element. A header and/or a footer can be added to a body element so that it can size and position itself accordingly.

Put the header and footer in front of the body element to allow the body element to resize itself to fit between any combination of header or footer. Using the .nav .nav-bar and the .nav-bar-head .nav-bar-foot hooks will give you a navbar wherever you need it.
